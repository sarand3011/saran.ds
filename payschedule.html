How the System Processes a PayGroup Change

The effective_date is not the exact move date — it is a lookup key the system uses to find the correct payrun in the new PayGroup.

Normal Flow — Monthly → Weekly, effective_date = Jan 10
PG2 Jan 1–30—Trimmed → Jan 1–9 effective_date Jan 10—Lookup → finds PG1 Jan 8–14 PG1 Jan 8–14—Updated → segment Jan 10–14
// Segment formula — eliminates overlap automatically
segment_start = max( dest_payrun_start, last_paid + 1, effective_date )
segment_end = dest_payrun_end

// Missing days → transition payrun
IF dest_payrun.status = APPROVED OR no payrun found
→ system creates transition payrun using new PayGroupVersion
